# Security Policy

## Supported Versions

Security updates are maintained on the latest main branch only.

## Reporting a Vulnerability

Please report security issues privately by email:

- security@yourdomain.com

Do not open a public GitHub issue for security vulnerabilities.

## What to Include

- Description of the vulnerability.
- Steps to reproduce.
- Affected files or pages.
- Suggested fix, if any.

## Security Practices

- Do not commit secrets or API keys.
- Use GitHub Secrets for deployment tokens.
- Enable Dependabot alerts.
- Enable secret scanning and push protection.
- Review dependencies regularly.

## Responsible Disclosure

We appreciate responsible disclosure and will respond as quickly as possible.
